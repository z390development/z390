z390 Portable Mainframe Assembler and Runtime Environment

The z390 portable mainframe assembler and runtime environment is a Java-based open source tool for writing, assembling, and running High Level Assembler programs.

z390 provides compatability with IBM's High Level Assembler at the source code level. The macros and other internals are fundamentally different.
Generated object code is portable to MVS 3.8, or to any other IBM operating system, provided you have a license to use the macro libraries for that operating system from IBM.

z390 offers the following base services:

Component Description
Macro processor HLASM is renowned for its extremely powerful macro processor, which allows you to define and use macros.
Assembler The assembler allows you to translate assembler source code into object code.
Linkage editor The linker allows you to combine object modules and transform them into a load module.
Runtime Environment The runtime environment allows you to execute and test a load module.
zCobol Compiler The zCobol component allows you to compile your Cobol programs.
Documentation Yes - we actually have some documentation!
Source Code Access z390 source code is managed in a public github repository

z390 also offers the following additional services:

Component Description
zCics The zCics component offers the ability to compile/assemble both Cobol and Assembler programs and run/test them in a Cics-like environment.
zVSAM Source-level compatible usage of data in a VSAM-like structure.
zSort The zSort program provides an out-of-the-box solution for sorting data.
Structured macro code A non-compatible extension that allows you to use structured coding elements at the macro code level.
Options files z390 supports a large number of options to control its behavior. Files can be chained to avoid duplication.
Scripted debugging The scripted debugging facility allows you to script your run-time tests, but can also be used for on-line debugging.
SOA support Provides support for Service Oriented Architecture.
Tracing z390 offers various traces to assist in debugging and analysis of run-time behavior.
Trace formatter Allows you to combine traces and format them for further analysis.

Macro processor

When in compatibility mode, the z390 macro processor behaves compatibly with IBM's HLASM macro facility. It can be used to process conditional assembly logic in open code, in stored macros, and in nested macros.

In z390 mode it offers options to allow non-compatible extensions, such as:

  • chkmac(0) - no additional checking. This is the default for chkmac.
  • chkmac(1) - check for missing AGO and AIF labels at macro load time.
  • chkmac(2) - as chkmac(1) and also check for non comment text after MEND.
  • mcall - list each inner macro call and exit on the assembly listing in addition to first level calls.
  • stats - Report statistics in a separate file.

Assembler

When in compatibility mode, the z390 assembler behaves compatibly with IBM's HLASM assembler. Use of macros, copy members, and the program code stack (AINSERT) are fully supported.

In z390 mode it offers options to allow non-compatible extensions, such as:

  • assist - allow use of ASSIST mnemonics.
  • chksrc(0) - no additional checking. This is the default for chksrc.
  • chksrc(1) - check assembler source files for any non-ASCII characters.
  • chksrc(2) - check assembler source files, macros, and copy members for any non-ASCII characters.
  • chksrc(3) - as chksrc(2) and also check for out of sequence characters in columns 73-80 or any non-blank characters beyond column 80.
  • init - Initialize all registers to hex x'F4', all memory to hex x'F5', and all uninitialized load module areas to x'F6'.
  • listcall - generate first level macro call and exit comments preceding assembler lines with "+" for macro generated source.
  • listuse - list active USING definitions in the assembly listing file each time USING or DROP is processed.
  • reformat - reformat expanded source code to align all opcodes at column 10 and operands at column 16 if possible.
  • stats - Report statistics in a separate file.
  • zstrmac - support expansion of ZSTRMAC structured conditional macro code instructions.

Linkage Editor

When in compatibility mode, the z390 linkage editor behaves compatibly with IBM's binder.

In z390 mode it offers options to allow non-compatible extensions, such as:

  • mod - Generate raw code file from lz390 with .MOD suffix and no header or trailer and no RLDs.
  • stats - Report statistics in a separate file.

zCics:
  • cics - support EXEC CICS pre-processor expansion and constants and enable PROLOG and EPILOG options
