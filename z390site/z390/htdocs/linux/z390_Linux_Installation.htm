Installing Linux

This is a journal of my personal experience installing and using Linux to regression test z390.  This was my first experience using Linux after being a long time Windows user.

  • Purchased new Dell Inspiration laptop with 160 GB HD, 2 GB memory, Windows XP (it was time for an upgrade)

  • Used Partition Magic 8.0 for Windows to split HD into 4 - 40 GB partitions for XP, LX, data, backup

  • XP remained NTFS format, the other 3 partitions are VFAT (FAT32) for XP/LX sharing

  • Selected Ubuntu Linux as it is the most widely used per this Linux distribution analysis

  • Download Unbuntu 6.06 LTS CD ISO image from Ubuntu website

  • Burned ISO image to CD by double clicking on ISO file and using Windows wizard

  • Rebooted laptop and used F2 to change boot sequence to include CD

  • Booted the Ubuntu 6.06 LTS boot CD

  • Selected X Windows safe mode install (the default failed to install X Windows on laptop)

  • Ignore failure to moust FAT16 Dell support partition and press continue

  • Double click on "Install icon" on desktop and answer a few questions about defaults

  • Select free space on hard drive for creation of primary and swap Ubuntu Linux partitions

  • Reboot and select Ubuntu 6.06 LTS instead of Windows XP from new dual boot loader

  • Ubuntu 6.06 comes up in GUI mode with wireless mouse support and Internet access already configured using Firefox web browser

  • Click on "Updates" to download latest Ubuntu updates

  • For Linux new comers, the book "Beginning Ubuntu Linux From Novice to Professiona" by Keir Thomas may be useful reference while learning commands.

  • Ubuntu Linux 6.06 LTS is now ready to use along side Windows XP

    Upgrading Ubuntu Linux from 6.06 LTS to 8.04 LTS

    1. Boot Ubuntu 6.06 LTS
    2. Use system menu to go to system maintenance and select "Upgrade to 8.04 LTS"
      1. Install is automatic and takes under an hour on broadband connection.
      2. Eclipse 3.2 previously installed would not load correcly
      3. Downloaded Eclipse 3.5 from www.eclipse.org for Linus

    Install J2RE Java Runtime on Ubuntu Linux

    • Download and install Linux version of Sun Microsystems Java

      • Install the package version of Sun J2RE 1.5.0_13 available in predefined repository
        (This JRE can be used to run z390 v1.3.09b)

        • Boot Ubuntu 6.06 LTS

        • Select "Applications", "Add/Remove" menu item

        • Click show unsupported applications (lower right of add/remove dialog)

        • Enter search "Sun Java" and press enter

        • Click on Sun Java 5/0 Runtime

        • Click ok to install

        • Click enable for "Multiverse channel" when prompted

        • Read Sun J2RE license, check if ok and press forward

        • Enable J2RE as Firefox browser plugin for applets:

          • Start terminal from "Application", "Accessories", "Terminal" menu

          • Enter command:
            sudo apt-get install sun-java5-plugin

          • Enter password when prompted

        • Make JRE 1.5.0_13 the default for command line by running command:
          sudo update-alternatives --config java

          • Enter the desired java version selection number and press enter

      • Install latest update of Sun JDK 1.6.0u16 for use with Eclipse which includes JRE
        (This JDK can be used with Eclipse 3.5 to debug z390 v1.5.01+ java source and rebuild the z390.jar)

        • Download JDK Linux self extracting binary to desktop

        • Start terminal and enter following commands to extract files:
          cd Desktop
          chmod u+x jdk-6u4-linux-i586.bin
          ./jdk-6u4-linux-i586.bin

        • Delete existing local java directory if it already exists:
          sudo rm -r /usr/local/java

        • Move extracted directory from desktop to new local user application java directory:
          sudo mkdir /usr/local/java
          sudo mv jdk1.6.0_16 /usr/local/java/jdk1.6.0_16

        • Remove symbolic links if they already exist

          • sudo rm -r  /usr/local/bin/java

          • sudo rm -r /usr/local/bin/javac

          • sudo rm -r     /usr/local/bin/jar

        • Create symbolic links for command line references

          • sudo ln -s /usr/local/java/jdk1.6.0_16/bin/java  /usr/local/bin/java

          • sudo ln -s /usr/local/java/jdk1.6.0_16/bin/javac /usr/local/bin/javac

          • sudo ln -s /usr/local/java/jdk1.6.0_16/bin/jar     /usr/local/bin/jar

        • Create symbolic link for Firefox web brower java support

          • sudo ln -s /usr/local/java/jdk1.6.0_16/jre/plugin/i386/ns7/libjavaplugin_oji.so
                             /usr/lib/firefox-3.0.14/plugins/libjavaplugin_oji.so

      • Verify J2RE installed plugin for FireFox web browser

        • Stop and restart Firefox web browser (Blue Circle on menu bar)

        • Run Sun J2RE verification Applet  at this URL
          http://www.java.com/en/download/installed.jsp - the applet runs for a few seconds and then should display current J2RE version such as jre 1.6.0_16

      Thanks to Martin Ward for providing assistance on first Ubuntu application install.
     

    Installing z390 full release on Linux

    • Download z390 file image zip and install on Linux

      • Boot Ubuntu 8.04 LTS Linux

      • Start Firefox web browser

      • Open web page "www.z390.org"

      • Click on "Downloads"

      • Click on "z390_v1309_files.zip" to download to desktop

      • Right click on "z390_v1309_files.zip" and select "Extract here"

      • Right click on "z390_v1309_files.zip" and select "Move to trash"4

      • Move z390 extracted directory from desktop to /usr/local/lib/z390:

        • Start terminal from "System", "Accessories", "Terminal"

        • If z390 has been installed previously, remove the prior version:
          sudo rm -r /usr/local/lib/z390

        • Enter move command to move z390 directories and files:
          sudo mv ~/Desktop/z390 /usr/local/lib

        • Ensure access rights for all z390 directories and programs:
          sudo chmod -R a+rwx /usr/local/lib/z390

        • Define link to the z390.pl GUI startup perl script in z390 install directory:
          sudo ln -s /usr/local/lib/z390/z390.pl   /usr/local/bin/z390

        • Define link to the dos.pl dos command processor perl script in z390 directory:
          (use Linux terminal command "dos" to start z390 dos command processor - see below)
          sudo ln -s /usr/local/lib/z390/dos.pl   /usr/local/bin/dos

        • Define link to the cmd.pl dos bat file command processor perl script in z390 directory:
          (cmd.pl is executed instead of cmd.exe when z390 is running under Linux)
          sudo ln -s /usr/local/lib/z390/cmd.pl   /usr/local/bin/cmd.pl

      • Verify z390 installation by running IVP (Installation Verification Program):

        • Start dos command processor by entering command:
          dos

        • Start the IVP dos command IVP.BAT by entering:
          IVP

        • The above command will display the operating system, J2SE, and z390 versions, any install errors, and a return code.  If the return code is greater than 4, action must be taken to correct J2SE or z390 install to correct version.  The optional RT and MVS zip files must be correctly installed in z390 directory to eliminate all warnings and set return code 0.

    Installing z390 cumulative PTF on Linux
    • Download z390 cumulative PTF file image and install (z390 file image overlay is in same zip along with Windows InstallShield executable):
      • Find PTF to download on www.z390.org
      • Click on link and select "save to Desktop"
      • Right click PTF zip file on Desktop and select "Extract here"
      • Copy and paste z390 file image overlap onto existing z390 installed directory
      • Use chmod to reset z390 privaledges to rwx for all z390 files.

    Running z390 Graphical User Interface on Linux

    • Create desktop icon to start z390 GUI Interface

      • Right click on desktop and click on "Launcher"

      • Enter name "z390 GUI Interface"

      • Enter command "perl /usr/local/lib/z390/z390.pl"

      • press ok

      • Double click "z390 GUI Interface icon on desktop to start it

      • Click on "Help", "Releases" to see which version of Linux, z390, and J2RE is running

      • Enter following commands to assemble, link, and execute demo:
        cd demo
        asmlg DEMO

      • Enter the following command to view demo execution log file using gedit GUI editor:
        gedit DEMO.LOG

    Running z390 Command Line Interface on Linux

    • Start terminal from menu "System", "Accessories" to start command line interface

    • Enter the command "dos" to start the z390 dos bat file command processor

      • demo                            (run the DEMO.BAT demo)

      • asmlg demo/DEMO    (Assemble, link, and execute hello world demo)

      • exec   demo/DEMO trace con (Execute demo again with trace and output to console)

      • exit                               (end the dos bat file command shell)

    • gedit demo/DEMO.TRE (view the demo execution trace using gexit GUI text editor)

    Installing Eclipse on Linus

    • Install Eclipse 3.5 (This requires JRE 1.6.0_16 to avoid lost breakpoints in 13+14)

      • Boot Ubuntu 8/04 LTS

      • Use Foxfire browser to download Eclipse 3.5 tar for Linux to desktop

      • Expand tar to file directory on desktop

      • If eclipse has been installed previously, remove the prior version:
        sudo rm -r /usr/lib/eclipse

      • Enter move command to move z390 directories and files:
        sudo mv ~/Desktop/z390 /usr/lib

      • Ensure access rights for all eclipse directories and programs:
        sudo chmod -R a+rwx /usr/lib/eclipse

    • Create z390 source project directory in Eclipse workspace directory
      (I have created separate partition shared by Linux and Windows with z390 sources)

    • Set and verify current J2RE used by Eclipse

      • Double click desktop icon to start Eclipse

      • Select "Run", "Debug", "JRE" and browse to /usr/local/java/jre to select latest J2RE

      • Select "Project", "Clean" to rebuild classes from source with selected J2RE

    • Debug execution of existing regression test program z390/rt/test/testins2

      • Select "Run", "Debug", check stop in main, and set project to z390 and class to ez390

      • Select "arguments tab and enter path to z390/rt/test/testins2 plus desired options
        /media/sda5/work/z390/rt/test/testins2 trace notrap notiming con
        (The above source testing path is on a shared volume separate from Ubuntu primary volume, and it is separate from the z390 download and install version located in /usr/local/lib/z390 on the Ubuntu primary volume.  This allows independent testing of installs and source development.)

      • Click on "Debug" to start debugging with stop in main

      • Set any breakpoints on any z390 source code using "Run", "Toggle Breakpoint"

      • Click on green arrow to run to next breakpoint or end of program

      • View any trace output on console tab

     

    Install Adobe Acrobat to view PDF files on Linux

    • Install the package version of Adobe Acrobat for Linus defined in repository
      (This is required to view z390 documentation via "Help", "User Guide" menu)

      • Boot Ubuntu 6.06 LTS

      • Select "Applications", "Add/Remove" menu item

      • Click show unsupported applications (lower right of add/remove dialog)

      • Enter search "adobe" and press enter

      • Click on "Adobe Acrobat for reading PDF files"

      • Click ok to install
